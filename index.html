PlantDoc.

Локальная платформа для диагностики растений и оркестрации сценариев. n8n ведёт автоматизацию, Ollama работает на твоем компьютере, а внешний доступ можно аккуратно добрать через Cloudflared без лишней облачной связки.

local runtime

Смысл в локальности

PlantDoc работает рядом с данными и не заставляет двигать фото через лишние промежуточные сервисы. Это упрощает контроль и отладку.

  • Ollama запускается на твоем ПК.
  • n8n держит сценарии и вебхуки.
  • SQLite подходит для быстрых прототипов и журналов.
automation

n8n как управляющий слой

Вместо сложной ручной логики сайт опирается на n8n: там проще менять маршруты, добавлять ветки и собирать новые ответы.

  • Webhook для входящих запросов.
  • Хранение и выборка результатов.
  • Переиспользуемые workflow-связки.
deployment

Доступ без лишней магии

Vercel уже держит витрину, а Cloudflared можно добавить, когда понадобится стабильный внешний адрес для локального runtime.

  • Статический фронт без лишних зависимостей.
  • Локальный прокси для Ollama.
  • Отдельный канал для внешнего доступа.
request flow

Поток запроса читается с первого взгляда.

Когда картинка или событие попадает в систему, каждый шаг выполняет одну задачу и не тащит за собой лишний интерфейс.

Вход Запрос приходит из сайта, Telegram или другого внешнего источника, который уже умеет говорить с вебхуком.
Оркестрация n8n раскладывает данные по веткам, готовит payload и передает его в локальные сервисы без ручного вмешательства.
Inference Ollama отвечает за локальный анализ и выдает результат в том формате, который удобнее дальше обработать.
Результат UI или бот получают финальный ответ, а история остается в SQLite и может быть показана или скрыта по нужному сценарию.
