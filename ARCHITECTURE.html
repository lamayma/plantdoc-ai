⚡ Воркфлоу в n8n — пошагово

WF-1 · Анализ фото (основной)

1
Telegram Trigger
Слушает входящие сообщения с фото от пользователей бота.
Нода: Telegram Trigger → тип: message
2
Получить файл
Скачать фото с серверов Telegram через getFile API.
Нода: HTTP Request → GET Telegram file
3
Сохранить на диск
Записать фото в папку /uploads/plants/ с именем {timestamp}_{userId}.jpg
Нода: Write Binary File
4
GPT-4 Vision анализ
Отправить фото в OpenAI с системным промптом диагностики.
Нода: OpenAI → model: gpt-4o
5
Парсить JSON ответ
Извлечь: plant_name, disease, severity, confidence, recommendations.
Нода: Set → JSON parse
6
Сохранить в SQLite
INSERT в таблицы diagnoses + plants. Связать с user_id.
Нода: SQLite → INSERT
7
Ответить пользователю
Отправить отформатированный диагноз обратно в Telegram.
Нода: Telegram → sendMessage
8
IF: критический диагноз?
Если severity = critical → уведомить администратора.
Нода: IF → Telegram admin notify

WF-2 · API для сайта

1
Webhook GET /plant-photos
Сайт делает запрос каждые 30 секунд.
Нода: Webhook → GET method
2
SELECT из SQLite
Последние 20 диагнозов, сортировка по дате DESC.
Нода: SQLite → SELECT * FROM diagnoses ORDER BY created_at DESC LIMIT 20
3
Форматировать ответ
Собрать JSON с полями: id, img_url, name, status, diagnosis…
Нода: Code (JS, 10 строк)
4
Вернуть JSON
Respond to Webhook с массивом объектов.
Нода: Respond to Webhook

WF-3 · Загрузка с сайта

1
Webhook POST /plant-upload
Получает multipart/form-data с файлом от сайта.
Нода: Webhook → POST, Binary Data ON
2
→ WF-1 (тот же путь)
Передать файл в основной воркфлоу с флагом source: "website".
Нода: Execute Workflow → WF-1
3
Ответить сайту
{ status: "analyzing", id: "..." } — сайт покажет спиннер.
Нода: Respond to Webhook

SQL для создания таблиц (вставь в n8n SQLite Execute)

-- Выполни один раз при первом запуске

CREATE TABLE IF NOT EXISTS users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  telegram_id TEXT UNIQUE,
  username TEXT,
  first_name TEXT,
  total_analyses INTEGER DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  last_active DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS diagnoses (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER REFERENCES users(id),
  plant_name TEXT,
  disease TEXT,
  severity TEXT DEFAULT 'ok',
  confidence INTEGER DEFAULT 0,
  diagnosis_text TEXT,
  recommendations TEXT,
  img_path TEXT,
  img_url TEXT,
  source TEXT DEFAULT 'telegram',
  show_on_site BOOLEAN DEFAULT 1,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
